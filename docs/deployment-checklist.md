# AMTools Backend 部署检查清单

> 📋 **确保部署成功的完整检查清单**

## 🚀 部署前检查

### 本地环境
- [ ] Git 已安装并配置
- [ ] Node.js 18+ 已安装
- [ ] 项目代码已下载到本地
- [ ] 能够正常执行 `npm install` 和 `npm run build`

### 服务器环境
- [ ] 服务器可以通过 SSH 访问
- [ ] 服务器操作系统: Ubuntu 20.04+ / CentOS 8+ / Debian 11+
- [ ] 服务器配置: 最低 2GB RAM, 20GB 存储空间
- [ ] 服务器有公网 IP 地址
- [ ] 防火墙允许必要端口 (22, 80, 443, 3000)

### GitHub 配置
- [ ] GitHub 仓库已创建
- [ ] 代码已推送到 GitHub
- [ ] GitHub Container Registry 权限已启用
- [ ] SSH 密钥对已生成
- [ ] 公钥已添加到服务器
- [ ] 私钥已添加到 GitHub Secrets

## 🔧 部署过程检查

### 服务器软件安装
- [ ] Docker 已安装 (`docker --version`)
- [ ] Docker Compose 已安装 (`docker-compose --version`)
- [ ] 当前用户已添加到 docker 组
- [ ] Docker 服务正在运行 (`systemctl status docker`)

### 目录和权限
- [ ] 应用目录已创建 (`/opt/amtools`)
- [ ] 日志目录已创建 (`/opt/amtools/logs`)
- [ ] 上传目录已创建 (`/opt/amtools/uploads`)
- [ ] 备份目录已创建 (`/opt/amtools/backups`)
- [ ] 目录权限正确设置 (755)
- [ ] 目录所有者正确设置

### 环境配置
- [ ] 环境变量文件已创建 (`/opt/amtools/.env`)
- [ ] 环境变量文件权限设置为 600
- [ ] 数据库配置正确
- [ ] JWT 密钥已设置 (至少32字符)
- [ ] Redis 配置正确
- [ ] 邮件配置正确 (如果启用)

### GitHub Secrets
- [ ] `REMOTE_HOST` 已设置 (服务器IP)
- [ ] `REMOTE_USER` 已设置 (SSH用户名)
- [ ] `REMOTE_SSH_KEY` 已设置 (完整私钥内容)
- [ ] `REMOTE_PORT` 已设置 (如果不是22)

## 🚀 部署执行检查

### GitHub Actions
- [ ] 工作流文件存在 (`.github/workflows/deploy.yml`)
- [ ] 推送代码到 main 分支
- [ ] GitHub Actions 工作流开始执行
- [ ] Test 阶段通过 ✅
- [ ] Build and Push 阶段通过 ✅
- [ ] Deploy 阶段通过 ✅
- [ ] 整个工作流显示绿色 ✅

### 容器部署
- [ ] Docker 镜像成功拉取
- [ ] 旧容器成功停止和删除
- [ ] 新容器成功启动
- [ ] 容器状态为 "Up" (`docker ps`)
- [ ] 容器端口正确映射 (3000:3000)
- [ ] 容器重启策略设置为 "unless-stopped"

## ✅ 部署后验证

### 基本功能测试
- [ ] 健康检查接口响应正常
  ```bash
  curl http://localhost:3000/api/v1/health
  ```
- [ ] API 文档可以访问
  ```bash
  curl http://localhost:3000/api-docs
  ```
- [ ] 发送验证码接口正常
  ```bash
  curl -X POST http://localhost:3000/api/v1/auth/send-verification-code \
    -H "Content-Type: application/json" \
    -d '{"email": "test@example.com", "type": "register"}'
  ```

### 系统状态检查
- [ ] 容器资源使用正常 (`docker stats`)
- [ ] 应用日志无错误 (`docker logs amtools-backend`)
- [ ] 系统资源充足 (`free -h`, `df -h`)
- [ ] 网络端口正常监听 (`netstat -tlnp | grep :3000`)

### 数据库连接
- [ ] 数据库连接正常
- [ ] 数据库表结构正确创建
- [ ] 数据库权限配置正确

### 缓存系统
- [ ] Redis 连接正常 (如果使用)
- [ ] 内存缓存工作正常 (如果使用)
- [ ] 缓存键前缀正确设置

### 邮件服务
- [ ] SMTP 连接正常 (如果启用)
- [ ] 邮件模板加载正常
- [ ] 验证码邮件发送成功
